Features, labels and instances
Let's take the example of a cat. Its features can be its tail and its ears for example. The features are the caracteristic of the object that you are studying. The label is the name that you associate with the combination of several features (= an instance). In this case it would be "cat".
    3 steps of Machine Learning
    Step 1: feature extraction / representation:
    You identify useful properties of instances (reminder: an instance is a vector of features). The features, also called attributes can be nominal, ordinal or numeric.
    Designing good features is complicated! There is a job called feature engineering.
    Step 2: training:
    Choose a hypothesis space (also called hypothesis class) that maps an instance to a specific class / label. The hypothesis is defined by parameters and learned by a classifier / model. g(x) = y with y being the label and x the instance.
    Machine learning is used when g(x) is unknown.
    Step 3: testing:
    Given a new instance x, use the classifier to predict a label inference. That way you can test the ability of your model to perform a task in a situation which has never been encountered before. Note that test instances should come from the same population.
    Assumption in ML: all training as well as test instances (and their labels) are independent and identically distributed (iid). A collection of random variables is iid if they all have the same probability distribution, and all are mutually independent.
